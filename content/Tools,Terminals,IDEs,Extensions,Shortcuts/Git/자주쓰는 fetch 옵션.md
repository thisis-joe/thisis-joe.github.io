### -p
```bash
git fetch -p origin
```
**하는 것**
- 원격 브랜치 정보 갱신  
- 원격에 새로 생긴 브랜치 정보 가져오기  
- 원격에서 삭제된 브랜치의 추적 정보 제거  
- 원격 커밋 정보 갱신

**안하는 것**
- 내 현재 브랜치 코드 변경 안 함  
- 작업 중인 파일 수정 안 함  
- 로컬 커밋 삭제 안 함  
- 로컬 브랜치 삭제 안 함  
- merge/rebase 안 함 

### main:main
```bash
git fetch origin main:main
```
- 현재 내 작업 브랜치에서 이동하지 않고, 원격 main의 최신 내용을 가져와서, 내 로컬 main 브랜치만 업데이트하기.
- 현재 브랜치가 `main`일 때는 사용하지 않는 것이 좋다.
- 주의
	- 현재 작업 중인 브랜치가 `main`이면 충돌/거절 될 수 있음.
	- 내 로컬 main에 원격에 없는 커밋이 있으면 충돌/거절될 수 있음.
