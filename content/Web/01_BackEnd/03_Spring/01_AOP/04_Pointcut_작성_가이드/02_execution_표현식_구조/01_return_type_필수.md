# 01_return_type_필수

## 상위 맥락
02_execution_표현식_구조 < 04_Pointcut_작성_가이드 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
AOP의 포인트컷(Pointcut) 필터링 지정자 중 가장 정교한 타겟팅을 수행하는 execution 표현식 구조 내에서 return_type(리턴 타입)은 대상 메서드를 식별하기 위해 반드시 작성해야 하는 필수 요소로 규정됩니다.

execution 표현식의 전체적인 구조는 execution(return_type(필수) Package+class method_name(필수)(parameter(필수)))의 형태를 가집니다.
이 아키텍처 맥락 속에서 return_type이 갖는 구체적인 특징과 작성 규칙은 다음과 같습니다.
### 1. 필수 입력 사항으로서의 역할 execution 표현식은 타겟 객체의 메서드 시그니처를 기반으로 조인 포인트(Join point)를 선별합니다. 이때 대상을 특정하는 패키지나 클래스명 부분은 생략이 가능할 수 있지만, 메서드의 반환 형태를 지정하는 return_type은 메서드 이름(method_name), 파라미터(parameter)와 함께 어드바이스 적용 여부를 결정짓는 절대적인 필수 기준으로 작용합니다.

### 2. 와일드카드를 통한 유연한 필터링과 부정 연산 수많은 비즈니스 메서드들의 반환 타입이 각기 다르기 때문에, AOP는 이를 유연하게 묶어서 필터링할 수 있는 와일드카드 및 기호 기능을 제공합니다.
`*` (별표): 타입에 무관함을 의미합니다.
즉, 대상 메서드의 반환값이 int이든, String이든, 혹은 반환값이 없는 void이든 상관없이 타겟으로 지정하겠다는 범용적인 필터링 조건입니다.

**! (느낌표): 특정 타입에 대한 부정(NOT)**을 의미합니다.
예를 들어 !String이나 !void와 같이 작성하면, 해당 타입을 반환하는 메서드만 철저하게 필터링망에서 제외(배제)하는 방식으로 타겟을 좁혀나갈 수 있습니다.

### 3. 레퍼런스 타입 지정 시의 엄격한 패키지 명시 규칙 return_type(및 parameter) 자리에 스칼라 타입(기본 자료형)이 아닌 레퍼런스(참조) 타입을 명시하여 대상을 필터링할 경우에는, 반드시 해당 객체의 '패키지 이름'까지 모두 포함하여 전체 경로를 작성해야 합니다.
예를 들어 특정 리스트 타입을 반환하는 메서드만을 타겟팅하고 싶다면, 단순히 List가 아니라 public java.util.List<com.example.MyClass>와 같이 패키지명이 완전히 포함된 형태로 명시해야 스프링 컨테이너가 런타임에 이를 정확한 타입으로 인식하고 검토할 수 있습니다.

정리하면 execution 표현식 구조 내에서 return_type은 대상 메서드의 시그니처를 1차적으로 걸러내는 강력하고 필수적인 필터링 관문입니다.
개발자는 프록시가 가로챈 수많은 메서드 중에서 *나 ! 같은 기호를 통해 광범위하게 타겟을 지정하거나, 패키지명이 포함된 정확한 레퍼런스 타입을 명시함으로써, 특정 형태의 데이터를 반환하는 비즈니스 로직에만 정밀하게 부가 기능을 결합(Weaving)시키는 고도의 제어력을 확보하게 됩니다.
