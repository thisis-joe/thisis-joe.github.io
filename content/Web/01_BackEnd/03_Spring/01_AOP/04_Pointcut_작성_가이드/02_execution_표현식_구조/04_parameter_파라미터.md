# 04_parameter_파라미터

## 상위 맥락
02_execution_표현식_구조 < 04_Pointcut_작성_가이드 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
AOP의 Pointcut 지정자 중 가장 정교한 타겟팅을 수행하는 execution 표현식은 '리턴 타입', '패키지 및 클래스', '메서드 이름', 그리고 '파라미터'라는 4가지 구역으로 구조화되어 있습니다.

파라미터 영역은 메서드가 전달받는 인자의 타입과 개수를 기준으로 대상을 좁히는 구역입니다.
리턴 타입과 메서드 이름으로 기본 대상을 좁힌 뒤, 파라미터 조건을 통해 오버로딩된 메서드까지 더 세밀하게 구분할 수 있습니다.

execution 표현식 구조 내에서 이 파라미터(parameter) 조건이 어떻게 작성되고 동작하는지 구체적인 원리를 살펴보면 다음과 같습니다.
### 1. 이름 없는 타입 기반의 선별 및 선택적 작성 파라미터 영역을 작성하여 타겟을 좁힐 때는, 실제 메서드에 선언된 파라미터의 변수 이름은 배제하고 오직 '타입'만을 명시하여 필터링 기준으로 삼는 선택적 구역으로 활용됩니다.

### 2. 와일드카드를 통한 유연한 시그니처 매칭 타겟 객체들이 가지는 다양한 파라미터 개수와 타입을 모두 하드코딩할 수 없으므로, 파라미터 구역에는 주로 다음과 같은 와일드카드가 배치되어 유연한 매칭을 수행합니다.
`*` (별표): 파라미터의 구체적인 타입에는 무관하지만, 반드시 **'정확히 1개의 파라미터'**를 요구하는 메서드 시그니처를 매칭할 때 사용합니다.
`..` (점 두 개): 파라미터의 타입과 개수에 전혀 제한을 두지 않고, **'0개 이상의 파라미터'**를 모두 허용하는 가장 포괄적인 형태의 대체 기호로 사용됩니다.

### 3. 레퍼런스 타입 지정 시의 주의사항 파라미터나 리턴 타입 조건에 스칼라(기본형) 타입이 아닌, List나 사용자 정의 객체 같은 레퍼런스(참조) 타입을 직접 명시하여 필터링하고자 할 때는 반드시 java.util.List와 같이 패키지 이름을 포함한 전체 경로(Full path)를 적어주어야만 시스템이 정확히 인식할 수 있습니다.

정리하면 execution 표현식 아키텍처 내에서 파라미터(parameter) 선별 기능은 필수 요소(리턴 타입, 메서드명)로 좁혀진 대상들 중에서, 오버로딩(Overloading)된 메서드들을 정밀하게 구별해 내거나 넓게 포괄할 수 있게 해주는 마이크로 필터링의 완성 단계입니다.
개발자는 이 선택적 구역에 *나 ..와 같은 와일드카드를 적절히 결합함으로써, 핵심 비즈니스 로직의 코드를 전혀 훼손하지 않고도 파라미터 구조의 다형성을 완벽하게 통제하며 부가 기능(Advice)을 엮어낼(Weaving) 수 있습니다.
