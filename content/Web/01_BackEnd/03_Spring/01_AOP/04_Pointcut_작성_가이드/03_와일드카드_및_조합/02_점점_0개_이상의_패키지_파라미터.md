# 02_점점_0개_이상의_패키지_파라미터

## 상위 맥락
03_와일드카드_및_조합 < 04_Pointcut_작성_가이드 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
AOP(관점 지향 프로그래밍)의 Pointcut(포인트컷) 작성 문법에서 와일드카드는 대상(Target)의 시그니처를 일일이 하드코딩하지 않고 패턴 매칭을 통해 범용적이고 유연한 필터링 규칙을 정의하기 위한 필수적인 도구입니다.
그중에서도 ..(점 두 개) 와일드카드는 패키지 계층의 깊이나 파라미터의 개수와 같이 '길이가 가변적인 구조'를 '0개 이상'이라는 조건으로 일괄 대체하는 강력한 포괄성을 지닙니다.

'와일드카드 및 조합'이라는 더 큰 맥락 속에서 .. 기호가 어떻게 동작하고 결합되는지 논리적 인과관계에 따라 살펴보면 다음과 같습니다.
### 1. 와일드카드 시스템 내에서의 역할 분담 Pointcut의 execution 지정자는 대상을 정밀하게 필터링하기 위해 여러 와일드카드를 조합합니다. *(별표)가 0개 이상의 문자열을 대체하거나 파라미터 부분에서 '정확히 1개의 파라미터' 등 단일 요소를 포괄하고, !가 특정 조건의 부정을 나타낸다면..는 시그니처 내에서 '연속된 구조의 생략과 포괄'을 전담합니다.

### 2. 패키지 경로에서의 조합 (0개 이상의 하위 패키지 대체) Pointcut 표현식의 패키지 및 클래스(Package + class) 선언부에 ..가 조합될 경우, 이는 '0개 이상의 하위 패키지'를 모두 대체하여 포함하겠다는 의미로 동작합니다.

예를 들어 특정 도메인 산하에 수많은 하위 패키지가 존재할 때, 이들의 경로를 개별적으로 명시하지 않고 ..를 조합함으로써 아키텍처의 특정 계층 전체를 한 번에 묶어낼 수 있습니다.
단, 작성 규칙상 이 .. 와일드카드는 패키지 경로의 맨 처음 부분에는 등장할 수 없다는 구조적 제약이 존재합니다.

### 3. 파라미터 구조에서의 조합 (타입/개수 무관 0개 이상의 파라미터 대체) Pointcut 표현식의 파라미터(parameter) 괄호 내부에 ..가 사용되면, 이는 파라미터의 타입이나 개수에 전혀 구애받지 않고 '0개 이상의 파라미터'를 모두 허용하겠다는 극단적인 유연성을 부여합니다.

이는 '파라미터의 타입에 무관하게 정확히 1개의 파라미터 대체'만을 허용하는 * 와일드카드와 명확한 구조적 대조를 이룹니다.

이러한 속성 덕분에 파라미터가 아예 없는 메서드부터 여러 개의 복합적인 파라미터를 전달받는 메서드까지 모두 단 하나의 Pointcut 규칙으로 수용할 수 있게 해줍니다.

정리하면 .. 와일드카드는 * 등의 다른 기호와 결합하여 Pointcut의 필터링 범위를 입체적으로 통제하는 핵심 연산자입니다.
개발자는 리턴 타입, 패키지 깊이, 클래스명, 파라미터 구조 등 각 시그니처 요소에 ..와 *를 적재적소에 조합함으로써 정밀하면서도 포괄적인 필터망을 구축할 수 있습니다.
이러한 와일드카드들의 유기적인 조합 논리 덕분에, 향후 대상 객체(Target)의 내부 구조(패키지 이동, 파라미터 추가 등)가 변경되더라도 AOP의 위빙(Weaving) 설정이 깨지지 않는 강력한 결합도 감소 효과를 얻게 됩니다.
