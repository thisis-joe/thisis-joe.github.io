# 03_bean_빈_이름_기준

## 상위 맥락
01_지정자_종류 < 04_Pointcut_작성_가이드 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
AOP 아키텍처에서 **Pointcut(포인트컷)**은 수많은 Join point(메서드 실행 시점) 중에서 Advice(부가 기능)를 적용할 대상을 걸러내는 필터 역할을 수행하며, 이 대상을 선별하는 기준을 구체적으로 정의하기 위해 **다양한 종류의 지정자(Designator)**를 사용합니다.

Pointcut 지정자 종류라는 더 큰 맥락 속에서 bean (빈 이름 기준) 지정자가 갖는 아키텍처적 의미와 동작 원리를 분석하면 다음과 같습니다.
### 1. 지정자(Designator) 종류의 다각화와 필터링 차원 스프링 AOP는 개발자가 의도한 타겟(Target)을 정확하게 조준하기 위해 다음과 같이 필터링의 기준(차원)을 나눈 여러 지정자를 제공합니다.

execution: 메서드의 리턴 타입, 패키지, 클래스, 메서드명, 파라미터 등 메서드 시그니처 전체를 분석하여 가장 정밀하게 타겟을 선별합니다.

within: 클래스나 패키지라는 물리적 범위를 기준으로 일괄 선별합니다.

@annotation 및 @within: 객체나 메서드에 부착된 특정 애너테이션의 존재 여부를 기준으로 선별합니다.

그리고 bean 지정자는 이들과 완전히 다른 접근법을 취하여, 오직 스프링 컨테이너에 등록된 빈(Bean)의 '이름' 자체를 기준으로 대상을 식별합니다.

### 2. bean 지정자의 동작 원리와 인과관계 bean 지정자는 대상 메서드의 구조적인 정보(시그니처)나 자바 패키지 경로를 분석하는 대신, 스프링의 프레임워크 핵심인 IoC 컨테이너가 각 객체를 관리할 때 부여하는 고유한 식별자(빈 이름)에 직접적으로 의존하여 동작합니다.

구체적인 작성 방식: @Before("bean(basicMemberDao)")와 같이, Pointcut 표현식 내부에 타겟팅할 대상 빈의 정확한 등록 이름을 명시합니다.

인과관계에 따른 실행 흐름: 스프링 컨테이너가 초기화되며 애플리케이션의 빈들을 생성할 때, 현재 생성 중인 빈의 이름이 bean() 지정자에 명시된 이름(basicMemberDao)과 일치하는지 평가합니다.
일치한다고 판별되면 스프링은 해당 대상을 프록시(Proxy)로 감싸고, 그 빈 객체 내부에서 실행되는 모든 메서드(Join point)에 지정된 부가 기능(Advice)이 개입하도록 결합(Weaving)시킵니다.
정리 Pointcut을 위한 지정자 생태계 내에서, execution이 자바 메서드의 '구조와 형태'를 분석하는 가장 보편적이고 정교한 필터라면, bean 지정자는 철저하게 스프링(Spring) 프레임워크의 빈 관리 생명주기에 밀착되어 동작하는 '특정 인스턴스' 타겟팅 전용 필터입니다.

개발자는 이 bean 지정자를 활용함으로써, 대상 클래스의 길고 복잡한 패키지 경로(com.ssafy..dao..* 등)나 파라미터 규칙을 와일드카드 기호로 어렵게 조합할 필요가 없어집니다.
단지 타겟팅하고자 하는 빈의 등록 이름 하나만 명시하는 것만으로 해당 객체의 모든 비즈니스 흐름을 직관적으로 통제할 수 있게 되어, 포인트컷 표현식의 가독성과 관리 효율을 극대화할 수 있습니다.
