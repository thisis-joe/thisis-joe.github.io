# 04_annotation_특정_애너테이션_기준

## 상위 맥락
01_지정자_종류 < 04_Pointcut_작성_가이드 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
AOP(관점 지향 프로그래밍) 생태계에서 **Pointcut(포인트컷) 지정자(Designator)**는 수많은 조인 포인트(메서드 실행 지점) 중에서 부가 기능을 엮어낼(Weaving) 정확한 대상을 필터링하기 위해 사용되는 다양한 기준 규격들을 의미합니다.

이러한 지정자의 종류로는 메서드 시그니처를 기반으로 하는 execution, 패키지나 클래스의 범위를 기준으로 하는 within, 스프링 컨테이너가 관리하는 빈(Bean)의 이름을 식별하는 bean 등이 존재합니다.
이 다양한 지정자 체계 속에서 @annotation은 메서드의 물리적인 위치나 이름이 아닌 '부여된 메타데이터(애너테이션)'를 기준으로 대상을 선별하는 매우 독자적이고 유연한 필터링 방식입니다.

@annotation 지정자가 갖는 역할과 전체 아키텍처 내에서의 인과관계는 다음과 같습니다.
### 1. 메타데이터 기반의 정밀한 타겟팅 @annotation 지정자는 이름 그대로 '특정 애너테이션이 선언된 메서드'만을 정확하게 찾아내어 부가 기능(Advice)의 적용 대상으로 결정합니다.
예를 들어, @Before("@annotation(org.springframework.transaction.annotation.Transactional)")와 같이 표현식을 작성하면, 시스템은 메서드의 패키지 경로나 리턴 타입, 파라미터 구조 등을 전혀 따지지 않고 오직 @Transactional 애너테이션이 마킹되어 있는 메서드만을 선별하여 실행을 가로챕니다.

### 2. @within 지정자와의 적용 범위(Scope) 차이 스프링 AOP는 애너테이션을 기준으로 삼는 또 다른 지정자로 @within을 제공하는데, 이 둘은 타겟을 좁히는 범위와 방식에서 엄격한 차이를 가집니다.
@within은 특정 애너테이션(예: @Service)이 '빈(Bean) 클래스 레벨'에 적용되어 있는지 확인하고, 기준을 충족하면 해당 빈 내부에 존재하는 모든 메서드를 일괄적으로 Advice 적용 대상으로 묶어버립니다.
반면, @annotation은 철저하게 메서드 레벨을 검사하여, 해당 애너테이션이 명시적으로 선언된 특정 메서드 단건에만 부가 기능을 적용하는 높은 정밀도를 보장합니다.

정리하면 지정자 종류라는 더 큰 맥락에서 볼 때, @annotation은 AOP 설정과 물리적 코드 구조 사이의 결합도를 획기적으로 낮춰주는 핵심 지정자입니다.

가장 범용적으로 쓰이는 execution 지정자는 패키지 경로, 클래스명, 메서드명 등 코드의 물리적인 구조(Signature)에 강하게 의존하여 대상을 필터링합니다.
따라서 패키지 구조가 개편되거나 메서드 명명 규칙이 바뀌면 Pointcut 표현식도 함께 수정되어야 하는 취약점이 존재합니다.
그러나 @annotation을 지정자로 활용하면 이러한 물리적 구조 제약에서 완전히 벗어날 수 있습니다.
개발자는 로깅이나 권한 체크 등 횡단 관심사가 필요한 비즈니스 메서드 위에 단순히 특정 애너테이션을 태깅(Tagging)해 두기만 하면 되므로, 코드의 리팩토링이나 패키지 이동과 무관하게 AOP가 언제나 안정적으로 동작하는 선언적이고 유연한 아키텍처를 완성할 수 있습니다.
