# 01_execution_메서드_시그니처_기준

## 상위 맥락
01_지정자_종류 < 04_Pointcut_작성_가이드 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
AOP(관점 지향 프로그래밍) 아키텍처에서 Pointcut은 수많은 실행 가능 지점(Join point) 중 부가 기능(Advice)을 적용할 대상을 선별해 내는 필터입니다.
이 대상을 정확히 어떤 기준으로 필터링할지 결정하는 문법적 기준을 **지정자(Designator)**라고 부르며, 그중에서도 execution 지정자는 대상의 '메서드 시그니처'를 분석하여 가장 정교하고 세밀하게 타겟을 선별하는 핵심적인 방식입니다.

지정자의 전체적인 맥락 속에서 execution이 어떻게 동작하고 어떤 구조적 가치를 지니는지 구체적으로 분석해 드리겠습니다.
### 1. Pointcut 지정자(Designator) 종류의 전체적 맥락 스프링 AOP는 타겟을 선별하기 위해 다양한 종류의 지정자를 제공합니다.

within: 클래스나 패키지 단위로 묶어 그 내부의 모든 메서드를 선별합니다.

bean: 스프링 컨테이너에 등록된 빈(Bean)의 이름을 기준으로 식별합니다.

@within 및 @annotation: 타겟 빈이나 메서드에 특정 애너테이션이 부여되었는지 여부를 기준으로 선별합니다.

이러한 지정자들은 '특정 공간(패키지/클래스)'이나 '특정 속성(이름/애너테이션)'을 기준으로 다소 거시적인 필터링을 수행합니다.

### 2. execution: 메서드 시그니처를 활용한 마이크로(Micro) 필터링 반면, execution 지정자는 타겟 객체가 가진 각각의 메서드가 고유하게 띠고 있는 형태, 즉 '시그니처(리턴 타입, 패키지+클래스 경로, 메서드명, 파라미터 구조)'를 필터링의 절대적 기준으로 삼습니다.
이 방식은 앞선 지정자들과 비교할 때 가장 미시적이고 정밀한 타겟팅을 가능하게 합니다.

execution 지정자는 필수 입력 사항인 '리턴 타입'과 '메서드 이름'을 바탕으로 다음과 같은 와일드카드를 결합하여 유연하고 강력한 필터링 규칙을 작성합니다.

리턴 타입 (필수): *를 사용하여 타입에 무관하게 적용하거나, !를 사용하여 특정 반환 타입(!String, !void 등)을 가진 메서드만 배제할 수 있습니다.

패키지 및 클래스명: ..를 사용하면 명시된 패키지와 그 아래에 있는 '0개 이상의 하위 패키지'를 단번에 대체하여 연결할 수 있습니다.

메서드 이름 (필수): *를 사용하여 0개 이상의 문자열을 대체할 수 있으므로, 특정 단어로 시작하거나 끝나는 메서드(예: set*, *Detail)를 일괄적으로 조준할 수 있습니다.

파라미터 구조: *는 파라미터의 타입에 무관하게 '정확히 1개의 파라미터'가 존재해야 함을 의미하며..는 타입과 개수에 아무런 제약 없이 '0개 이상의 파라미터'를 모두 허용하겠다는 의미로 사용됩니다.

정리하면 Pointcut 지정자 종류 내에서 execution의 '메서드 시그니처 기준' 필터링은 개발자에게 외과 수술과도 같은 정밀한 제어권을 제공합니다.

거시적인 지정자(within, bean)만으로는 "회원 관리 클래스 중에서 'String을 반환'하고 '두 개의 파라미터'를 받는 특정 메서드에만 로깅을 적용해라"와 같은 복잡한 요구사항을 충족시킬 수 없습니다.
하지만 execution 지정자는 이 모든 시그니처 요소를 정규 표현식처럼 섬세하게 분해하여 필터링하므로, 개발자는 비즈니스 코드를 전혀 수정하지 않은 상태에서도 시스템 전반의 핵심 동작 흐름을 완벽하고 유연하게 타겟팅하여 횡단 관심사를 엮어낼(Weaving) 수 있습니다.
