# 02_Spring_AOP에서는_메서드_실행_시점으로_제한

## 상위 맥락
04_Join_point < 02_AOP_주요_용어 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
Spring AOP에서 Advice가 적용될 수 있는 지점은 메서드 실행 시점으로 제한됩니다.
이 제한은 Spring AOP가 프록시 기반으로 동작하기 때문에 발생합니다.
프록시는 외부에서 들어오는 메서드 호출을 가로채는 방식으로 동작합니다.
따라서 객체 내부의 필드 값 변경이나 생성자 호출 과정에는 개입할 수 없습니다.
이 구조 때문에 Spring AOP의 Join point는 이론적인 AOP보다 좁은 범위를 가집니다.
이 제한은 Advice 타입의 기준도 결정합니다.
@Before는 타겟 메서드 실행 전에 동작합니다.
@AfterReturning은 타겟 메서드가 정상 종료된 뒤 동작합니다.
@AfterThrowing은 타겟 메서드 실행 중 예외가 발생했을 때 동작합니다.
@After는 성공과 실패 여부와 관계없이 종료 시점에 동작합니다.
@Around는 타겟 메서드 실행 전후를 감싸고 proceed() 호출을 통해 실행 여부까지 제어합니다.
정리하면 Spring AOP에서 메서드 실행 시점으로 제한된다는 말은, 프록시가 가로챌 수 있는 외부 호출 지점만 AOP 적용 대상이 된다는 의미입니다.
