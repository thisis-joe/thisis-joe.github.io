# 02_Advice를_적용할_대상_결정_필터

## 상위 맥락
05_Pointcut < 02_AOP_주요_용어 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
AOP(관점 지향 프로그래밍)의 아키텍처에서 **Pointcut(포인트컷)**은 시스템 내의 수많은 실행 지점 중 'Advice(부가 기능)를 적용할 대상을 결정하는 필터(표현식)' 역할을 수행하는 핵심 요소입니다.

이 필터링 메커니즘이 AOP의 전체 구조 속에서 가지는 구체적인 의미와 동작 원리는 다음과 같습니다.
### 1. 필터로서의 Pointcut과 Join point의 인과관계 스프링 AOP(Spring AOP) 환경에서는 프록시(Proxy)를 기반으로 동작하기 때문에, 부가 기능인 Advice가 개입할 수 있는 후보 지점, 즉 조인 포인트(Join point)는 철저히 '메서드 실행 시점'으로 제한됩니다.
하지만 애플리케이션 내에 존재하는 모든 비즈니스 메서드에 로깅이나 트랜잭션과 같은 부가 기능을 무작위로 적용할 수는 없습니다.
따라서 **무수히 많은 Join point(메서드 호출)들 중에서 실제로 Advice 로직을 엮어낼(Weaving) 타겟(Target) 메서드만을 정확하게 선별해 내는 거름망(Filter)**이 필요한데, 시스템은 이 대상을 결정하기 위해 Pointcut 표현식을 필터로 사용합니다.

### 2. 대상을 결정하는 정교한 필터링 지시자(Designator) Pointcut은 주로 @Before, @AfterReturning 등 Advice를 선언하는 애너테이션의 value 속성 값으로 설정되어 대상의 호출 여부를 결정합니다.
대상을 정확히 필터링하기 위해 다음과 같은 다양한 지정자 방식을 지원합니다.
execution: 메서드의 리턴 타입, 패키지 및 클래스명, 메서드명, 파라미터 등 메서드 시그니처 전체를 기반으로 가장 정교한 필터링을 수행합니다.
필터링 시 *(타입 무관)..(0개 이상의 하위 패키지 또는 파라미터)와 같은 와일드카드를 조합하여 유연하게 대상을 지정할 수 있습니다.

within: 특정 클래스나 패키지 단위로 범위를 좁혀 필터링합니다.

bean: 스프링 컨테이너에 등록된 빈(Bean)의 이름을 기준으로 대상을 필터링합니다.

@annotation 및 @within: 타겟 객체의 메서드나 클래스에 특정 애너테이션(예: @Transactional)이 선언되어 있는지를 필터 기준으로 삼아 Advice 적용 대상을 판별합니다.

### 3. 복합 필터링 연산 및 Pointcut의 재사용 Pointcut은 단일 조건에 그치지 않고, &&(and), ||(or), !(not)와 같은 논리 연산자를 결합하여 여러 개의 필터 조건을 다각도로 연결할 수 있습니다.
또한, 하나의 필터 조건(표현식)이 여러 Advice에서 공통적으로 필요할 경우, @Pointcut 애너테이션을 이용해 빈 메서드로 선언해두면(Named Pointcut), 다른 Advice들이 해당 필터의 이름(메서드 id)을 호출하는 방식으로 손쉽게 필터 조건을 재사용할 수 있어 코드의 응집도를 높여줍니다.

정리 결과적으로 AOP의 작동 체계 내에서 Advice가 "어떤 부가 기능을, 언제(When/What) 수행할 것인가"를 정의한다면, Pointcut이라는 필터는 그 기능이 "정확히 어디에(Where) 개입할 것인가"를 통제하는 결정권자입니다.
이 정교한 필터링 메커니즘 덕분에 핵심 비즈니스 로직(Target)의 코드를 단 한 줄도 수정하지 않고도, 개발자가 의도한 정확한 위치의 메서드에만 동적으로 부가 기능을 결합(Weaving)시키는 완벽한 관점의 분리가 실현됩니다.
