# 02_횡단_관심_코드를_포함한_메서드

## 상위 맥락
03_Advice < 02_AOP_주요_용어 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
AOP(관점 지향 프로그래밍)의 주요 용어 체계에서 '횡단 관심 코드를 모듈화한 메서드'는 바로 어드바이스(Advice)를 의미합니다.

이러한 어드바이스(Advice)의 역할과 특징을 AOP의 전체적인 구조 속에서 살펴보면 다음과 같습니다.
### 1. 추상적 관점(Aspect)의 구체적 실체화 앞서 다룬 애스펙트(Aspect)가 '로깅', '보안' 등 시스템에 흩어진 횡단 관심사들을 묶어낸 추상적인 명칭(논리적 단위)이라면, **Advice는 그 관점을 시스템 내에서 실제로 동작시키기 위해 횡단 관심 코드를 작성해 둔 '구체적인 메서드'**를 의미합니다.
하나의 Aspect 내에는 이러한 Advice 메서드가 하나 이상 포함되어 실질적인 부가 기능을 수행하게 됩니다.

### 2. Advice 메서드의 구성 요소 이 메서드는 단순히 독립적으로 존재하는 것이 아니라, AOP 환경에서 자동으로 엮이기 위해 두 가지 핵심 요소로 구성됩니다.

메서드 바디(Body): 파라미터 조작, 로깅 출력, 실행 시간 측정 등 실제로 실행될 횡단 관심사 로직(코드) 자체입니다.

포인트컷(Pointcut) 정보: 이 메서드 로직을 수많은 비즈니스 로직 중 정확히 '어디에' 적용할 것인지를 결정하는 필터링 조건식입니다.

### 3. 타겟 메서드와의 실행 시점에 따른 5가지 타입 분류 Advice 메서드의 가장 두드러진 특징은 핵심 비즈니스 로직을 가진 타겟(Target) 메서드가 실행되는 시점(Join point)을 기준으로 언제 이 부가 기능 코드를 동작시킬지 엄격하게 규정된다는 점입니다.

@Before: 타겟 메서드가 본격적으로 실행되기 전에 동작하는 메서드입니다.

@AfterReturning: 타겟 메서드가 아무런 예외 없이 정상적으로 종료되어 값을 반환(return)한 직후에만 동작하는 메서드입니다.

@AfterThrowing: 타겟 메서드 내부 로직을 수행하던 중 예외가 던져졌을(throw) 때만 동작하여 일괄적인 예외 처리를 수행하는 메서드입니다.

@After: 타겟 메서드의 성공 여부나 예외 발생과 무관하게 무조건 마지막에 실행되는 메서드입니다.

@Around: 타겟 메서드 실행 전후의 모든 과정을 제어하는 가장 강력한 형태의 메서드입니다.
이 메서드 내부에서는 타겟 메서드를 직접 호출할 수 있으며, 타겟으로 전달되는 파라미터나 타겟이 뱉어낸 리턴 값을 완전히 조작하거나 대체할 수도 있습니다.

정리 결과적으로 기존 개발 방식에서는 핵심 비즈니스 로직을 처리하는 메서드 내부에 횡단 관심 코드가 어지럽게 섞여 있어야만 했습니다.
하지만 AOP 아키텍처에서는 이 횡단 관심 코드들을 'Advice'라는 독립된 메서드로 완벽하게 분리해 냅니다.
이후 스프링의 프록시(Proxy) 객체가 런타임 시점에 클라이언트의 요청을 가로채어, 지정된 타이밍(@Before, @After 등)에 맞게 이 Advice 메서드들을 대신 호출해 줌으로써 타겟 객체는 오로지 비즈니스 로직에만 전념할 수 있게 됩니다.
