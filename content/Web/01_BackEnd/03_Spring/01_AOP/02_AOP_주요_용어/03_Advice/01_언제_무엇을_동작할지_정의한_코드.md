# 01_언제_무엇을_동작할지_정의한_코드

## 상위 맥락
03_Advice < 02_AOP_주요_용어 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
AOP(관점 지향 프로그래밍)의 핵심 용어 체계에서 '언제 무엇을 동작할지 정의한 코드'는 바로 어드바이스(Advice)를 의미합니다.

Advice는 횡단 관심 코드를 모듈화한 구체적인 메서드이며, 내부적으로 '무엇을(advice code)' 수행할지와 '언제(advice type)' 개입할지를 모두 포함하여 구성됩니다.

이 두 가지 요소가 Advice 내에서 어떻게 결합하여 동작하는지 구체적인 인과관계와 흐름을 바탕으로 설명해 드리겠습니다.
### 1. 무엇을 동작할지 정의 (Advice Code) '무엇을' 동작할지는 Advice 메서드의 본문(body)에 작성된 실제 횡단 관심사(부가 기능) 로직을 의미합니다.
로깅 메시지를 출력하거나(log.debug), 클라이언트로부터 넘겨받은 파라미터를 암호화하거나, 클라이언트에게 반환할 데이터를 마스킹하는 등의 구체적인 기능이 여기에 해당합니다.
개발자는 비즈니스 로직(Target)에서 이러한 부가 코드들을 떼어내어 Advice 내부에 단 한 번만 정의함으로써 코드 중복을 제거하고 모듈화합니다.

### 2. 언제 동작할지 정의 (Advice Type) '언제' 동작할지는 조인 포인트(Join point)를 기준으로, 해당 부가 로직(Advice Code)이 타겟(Target) 메서드의 실행 과정 중 어느 타이밍에 개입할지를 결정하는 어노테이션 타입을 의미합니다.
Spring AOP는 타겟 메서드 실행 전후의 세밀한 제어를 위해 개입 시점을 5가지 타입으로 엄격하게 규정합니다.

@Before: 타겟 메서드가 호출되기 전에 부가 로직을 실행합니다.
타겟의 핵심 로직이 동작하기 전에 전달받은 파라미터(argument)를 가로채어 비밀번호를 암호화하는 등의 사전 조작이 가능합니다.

@AfterReturning: 타겟 메서드가 예외 없이 정상적으로 로직을 마치고 종료(return)된 경우에만 실행됩니다.
타겟이 반환한 결과값을 가로채어 민감 정보를 별표(*) 등으로 마스킹 조작한 뒤 반환할 수 있습니다.

@AfterThrowing: 타겟 메서드 내부의 로직을 실행하던 중 예외가 던져졌을(throw) 때만 개입하여 동작합니다.
이를 통해 특정 비즈니스 로직마다 일일이 예외 처리를 하지 않고도 일괄적인 예외 대책(예: 관리자에게 알림)을 수립할 수 있습니다.

@After: 타겟 메서드의 정상 완료나 예외 발생 여부와 무관하게, 메서드 실행이 완료되면 언제나(무조건) 실행됩니다 (마치 try~catch 구문의 finally와 동일한 역할을 수행합니다).

@Around: 타겟 메서드의 실행 전과 후의 모든 흐름을 제어하는 가장 강력하고 포괄적인 타입입니다.
Advice 내부에서 proceed()라는 메서드를 호출하여 원본 타겟의 실행 타이밍을 직접 결정하며, 필요하다면 타겟으로 들어가는 파라미터나 타겟이 내뱉는 리턴 값을 완전히 다른 객체로 대체해버리는 강력한 통제권을 가집니다.

정리 결과적으로 Advice 내부에 '무엇을'과 '언제'를 함께 정의한다는 것은, 부가 기능 스스로가 자신의 동작 시나리오를 완벽하게 통제하는 자립적인 모듈로 완성되었음을 의미합니다.

이러한 메커니즘을 통해 핵심 비즈니스 로직을 담당하는 타겟(Target) 객체는 자신에게 '언제 어떤 부가 기능이 개입하는지'를 전혀 알 필요가 없어집니다.
프록시(Proxy)가 런타임에 Advice의 정의된 시점(언제)을 파악하여 적절하게 부가 로직(무엇을)을 끼워 넣어주기 때문에, 타겟은 오직 데이터베이스 처리와 같은 순수한 본연의 목적에만 전념할 수 있는 완벽한 모듈화와 관심사의 분리가 달성됩니다.
