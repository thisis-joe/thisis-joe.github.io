# 02_AOP가_적용될_대상

## 상위 맥락
01_Target < 02_AOP_주요_용어 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
AOP(관점 지향 프로그래밍)에서 **Target(타겟)**은 **"핵심 비즈니스 로직을 포함하고 있는 빈(Bean) 객체이자, AOP(부가 기능)가 적용될 실제 대상"**을 엄격하게 지칭하는 기본 용어입니다.

Target의 구체적인 특징과 AOP 동작 원리 내에서의 맥락은 다음과 같습니다.
### 1. 순수한 핵심 비즈니스 로직의 보관소 Target은 로깅, 성능 측정, 보안 검사, 트랜잭션 처리와 같은 횡단 관심사(부가 로직)가 모두 배제된 순수한 비즈니스 로직(예: 사용자의 정보 저장 로직 등)만을 온전히 보유하고 있는 객체입니다.
개발자가 AOP를 도입하여 궁극적으로 집중하고자 하는 본질적인 코드가 바로 이 Target 내부에 존재합니다.
### 2. 프록시(Proxy)를 통한 간접적인 실행 대상 스프링 AOP(Spring AOP) 환경에서 Target에 부가 기능을 적용할 때는 원본 Target 객체의 코드를 직접 수정하지 않고, 대신 Target을 감싸는 '가짜 객체'인 프록시(Proxy)를 생성하여 간접적으로 동작합니다.

클라이언트(호출자)가 Target 객체의 기능을 사용하기 위해 스프링 컨테이너에서 빈을 요청(ctx.getBean())하면, 컨테이너는 원래의 Target 대신 부가 기능이 덧입혀진 Proxy 객체를 반환합니다.

결과적으로 클라이언트의 요청은 Proxy가 먼저 가로채어 받게 되며, Proxy가 필요한 부가 로직을 처리한 뒤 내부적으로 진짜 Target의 메서드를 호출해 주는 방식으로 AOP가 적용됩니다.

### 3. 어드바이스(Advice) 실행 시점의 기준점 분리된 부가 기능인 어드바이스(Advice)는 Target의 메서드(Target Method)가 실행되는 것을 기준으로 언제 부가 로직을 끼워 넣을지 결정합니다.

@Before: Target 메서드가 본격적으로 실행되기 전에 부가 기능(예: 파라미터 조작, 보안 검사)을 실행합니다.

@AfterReturning: Target 메서드가 성공적으로 실행을 마치고 정상적으로 값을 반환한 직후에 부가 기능(예: 반환값 마스킹)을 실행합니다.

@AfterThrowing: Target 메서드 실행 도중 예외가 발생했을 때만 부가 기능(예: 예외 대책 수립, 로그 기록)을 실행합니다.

@Around: Proxy가 직접 proceed()를 호출하여 실제 Target의 메서드 호출 자체를 전면 통제하며, Target 메서드 실행 전과 후의 모든 과정을 완벽하게 제어하거나 파라미터 및 반환값을 완전히 대체할 수도 있습니다.

요약하자면, AOP 아키텍처에서 Target은 흩어져 있던 여러 횡단 관심사(Aspect)들이 하나로 결합(Weaving)되는 최종 목적지입니다.
프록시 메커니즘 덕분에 Target 자체의 코드는 단 한 줄도 오염되거나 수정되지 않으면서도, 실행되는 시점에는 주변의 다양한 부가 기능(Advice)들로부터 완벽하게 지원을 받으며 동작할 수 있게 됩니다.
