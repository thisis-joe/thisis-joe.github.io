# 02_횡단_관심사의_모듈화_및_재사용

## 상위 맥락
03_AOP의_정의 < 01_AOP_기본_컨셉 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
**AOP(관점 지향 프로그래밍, Aspect Oriented Programming)**는 프로그램의 기능을 단순한 로직의 나열이 아니라, 필요한 개별 관점(Aspect)으로 바라보는 프로그래밍 기법입니다.
이러한 AOP의 정의를 관통하는 핵심적인 목적 중 하나가 바로 횡단 관심사의 모듈화 및 재사용입니다.
애플리케이션의 메서드는 크게 두 부분, 즉 '핵심 관심 사항(비즈니스 로직)'과 '부가 로직'으로 나뉩니다.
여기서 부가 로직을 횡단 관심 사항이라고 부르며, 로깅, 성능 측정, 보안, 암호화, 트랜잭션 처리와 같이 여러 메서드를 가로지르며 공통적으로 등장하는 기능들을 의미합니다.

이러한 횡단 관심사가 비즈니스 로직 곳곳에 중복되어 흩어져 있으면 코드가 복잡해져 유지 보수가 어려워지고, 초급 개발자가 모든 로직을 정상적으로 처리했는지 확신하기 힘들어지는 문제가 발생합니다.

AOP는 이 문제를 해결하기 위해 다음과 같은 과정을 거칩니다.
모듈화: 여러 코드에 흩어져 있는 횡단 관심사들을 추출하여 독립적인 모듈로 분리합니다.
AOP 용어에서는 이렇게 **모듈화된 횡단 관심사를 '애스펙트(Aspect)'**라고 부르며, **실제 횡단 관심 코드를 모듈화한 메서드를 '어드바이스(Advice)'**라고 정의합니다.

재사용 및 적용: 분리된 횡단 관심 모듈은 필요한 지점(Pointcut)에 동적으로 결합(Weaving)되어 여러 곳에서 재사용됩니다.

결론적으로, AOP의 정의 안에서 횡단 관심사의 모듈화와 재사용이 갖는 가장 큰 의의는 개발자가 부가적인 처리 로직에 신경 쓰지 않고 오직 "비즈니스 로직에 전념"할 수 있도록 만들어 준다는 것입니다.
이를 통해 코드의 중복을 줄이고 유지 보수성을 극대화하는 것이 AOP가 지향하는 궁극적인 목표입니다.
