# 02_코드_중복_및_유지보수_어려움

## 상위 맥락
01_기존_코드의_문제점 < 01_AOP_기본_컨셉 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
기존 코드의 핵심적인 문제점은 핵심 비즈니스 로직(핵심 관심사항)과 로깅, 트랜잭션 처리 등의 부가 기능(횡단 관심사항)이 하나의 메서드 안에 복잡하게 섞여 있다는 것입니다.
이러한 구조적 특징은 구체적으로 다음과 같은 코드 중복 및 유지보수의 어려움을 초래합니다.
광범위한 코드 중복 발생: 서비스 로직을 구현할 때 로깅, 성능 측정, 보안, 암호화, 트랜잭션 처리 같은 부가 기능들은 insert, update, delete 등 여러 메서드를 가로지르며 공통적으로 필요합니다.
이러한 횡단 관심사들이 결과적으로 **"알고 보면 모두 비슷한 코드들"**이라고 지적하며, 시스템 전반에 걸쳐 동일한 형태의 코드가 매번 반복해서 작성되는 심각한 중복 문제를 발생시킨다고 설명합니다.

유지보수의 어려움 및 개발자 부담 가중: 이처럼 반복되는 부가 로직이 수많은 시스템 코드 곳곳에 흩어져 있기 때문에, 코드에 수정 사항이 발생할 경우 일일이 찾아가며 고쳐야 하므로 유지보수가 매우 어려워집니다.
또한, 개발자는 본연의 비즈니스 로직뿐만 아니라 트랜잭션, 암호화 등 세부적으로 알아야 할 부가 기술들이 증가하여 부담이 커지며, 특히 초급 개발자의 경우 모든 로직이 정상적으로 처리되었는지 확신하기 어렵게 만듭니다.

결과적으로 AOP(관점 지향 프로그래밍)는 이러한 기존 코드의 근본적인 한계를 극복하기 위해 등장했습니다.
여러 곳에서 중복되는 횡단 관심사를 별도로 모듈화하여 재사용함으로써 코드 중복을 제거하고, 개발자가 복잡한 유지보수나 부가 기능 구현의 부담에서 벗어나 오직 비즈니스 로직에만 전념할 수 있도록 하는 것이 핵심 해결책입니다.
