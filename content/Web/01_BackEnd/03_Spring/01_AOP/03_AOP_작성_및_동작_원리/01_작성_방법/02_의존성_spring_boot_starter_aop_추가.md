# 02_의존성_spring_boot_starter_aop_추가

## 상위 맥락
01_작성_방법 < 03_AOP_작성_및_동작_원리 < Aspect_Oriented_Programming_AOP < Spring < Web

## 핵심 정리
스프링(Spring) 환경에서 AOP를 실제 코드로 구현하는 '작성 방법'의 가장 첫 번째 핵심 단계가 바로 spring-boot-starter-aop 의존성(Dependency)을 추가하는 것입니다.

이론적으로 설계한 관점(Aspect)과 어드바이스(Advice)를 실제 애플리케이션에서 동작시키려면, 스프링 컨테이너가 AOP 관련 애너테이션들을 인식하고 타겟 객체를 감싸는 프록시(Proxy)를 생성할 수 있도록 지원하는 기반 환경이 필수적입니다.
이를 위해 프로젝트의 빌드 설정 파일(예: pom.xml)에 <groupId>org.springframework.boot</groupId>와 <artifactId>spring-boot-starter-aop</artifactId> 의존성을 주입하게 됩니다.

이러한 의존성 추가를 시작으로, 실제 AOP를 코드 레벨에서 작성하는 프로세스는 다음과 같은 논리적인 흐름으로 전개됩니다.
관심사의 명확한 식별: 구현에 앞서 타겟 객체가 수행할 '핵심 관심사(예: 각각의 DAO에서 처리하는 핵심 로직)'와 덧붙여질 '횡단 관심사(예: DAO 메서드가 호출될 때 파라미터를 포함한 로그를 출력하는 작업)'를 철저하게 분리하여 기획합니다.

Aspect 클래스 생성 및 빈 등록: AOP 라이브러리가 활성화되었으므로, 부가 기능을 담을 클래스를 생성한 뒤 상단에 @Aspect 애너테이션을 부여하여 이 클래스가 하나의 '관점' 모듈임을 명시합니다.
동시에 스프링 프레임워크가 프록시를 통해 이 모듈을 관리하고 엮어낼 수 있도록 @Component 애너테이션을 추가하여 빈(Bean)으로 등록합니다.

Advice와 Pointcut의 구체화: Aspect 클래스 내부에는 실제로 수행할 부가 기능 로직을 담은 메서드(Advice code)를 작성합니다.
그리고 이 메서드 상단에 @Before와 같은 어드바이스 타입(실행 시점)과 execution(* com.ssafy..dao.*(.*))와 같은 포인트컷 표현식(적용 대상 필터링)을 선언하여 언제, 어디에 부가 기능을 동작시킬지 구체적으로 지시합니다.

결론적으로 AOP 작성 방법에 있어 spring-boot-starter-aop 의존성 추가는 단순한 라이브러리 임포트를 넘어, 개발자가 복잡한 프록시 객체 생성이나 위임 로직을 직접 구현하지 않고도 애너테이션 기반의 우아한 방식으로 핵심 로직과 부가 기능을 철저히 분리(모듈화)할 수 있도록 스프링의 강력한 AOP 메커니즘 엔진을 켜는 필수적인 진입점이라고 할 수 있습니다.
